# AGENTS.md

本项目用 **C 语言手工实现** L26 语言编译器，将 L26 源码编译为类 P-Code 指令，并在类 P-Code 虚拟机上解释执行。

下面是用 EBNF 描述的 L26 语言，所有的终结符用双引号括起表示。

```
<program> ::= <block>
<block> ::= "{" <decls> <stmts> "}"
<decls> ::= { <decl> }
<decl> ::= <type> <ID> ";"
<type> ::= "int" | "bool" | "set" (* set采用静态实现，最大支持200个集合元素 *)

<stmts> ::= { <stmt> }
<stmt> ::= <assign_stmt>
        | <if_stmt>
        | <while_stmt>
        | <io_stmt>
        | <block>
        | <set_op_stmt> (* 集合操作语句 *)

<assign_stmt> ::= <ID> "=" <expr> ";"
<if_stmt> ::= "if" "(" <bexpr> ")" <stmt> [ "else" <stmt> ]
<while_stmt> ::= "while" "(" <bexpr> ")" <stmt>
<io_stmt> ::= "write" <expr> ";" | "read" <ID> ";"
<set_op_stmt> ::= "add" <ID> <aexpr> ";" (* 向集合添加元素，不能破坏集合元素互异唯一性，元素顺序不影响集合 *)
                | "remove" <ID> <aexpr> ";" (* 从集合删除元素，不存在该元素则不改变原集合 *)

<expr> ::= <bexpr> | <aexpr> | <set_expr>

(* 算术表达式 *)
<aexpr> ::= <aterm> { ("+" | "-") <aterm> }
<aterm> ::= <afactor> { ("*" | "/") <afactor> }
<afactor> ::= <ID> | NUM | "(" <aexpr> ")"

(* 逻辑表达式 *)
<bexpr> ::= <bterm> { "||" <bterm> }
<bterm> ::= <bfactor> { "&&" <bfactor> }
<bfactor> ::= "true" | "false" | "!" <bfactor> | "(" <bexpr> ")" | <rel> | <set_test>

(* 关系运算 *)
<rel> ::= <aexpr> ("<" | "<=" | ">" | ">=" | "==" | "!=") <aexpr>

(* --- SET 集合相关文法 --- *)
<set_expr> ::= "{" [ <aexpr> { "," <aexpr> } ] "}" (* 集合字面量：{1,2,3} *)
             | <ID> "union" <ID> (* 并集 *)
             | <ID> "inter" <ID> (* 交集 *)

<set_test> ::= <aexpr> "in" <ID> (* 成员检查 *)
             | "isempty" "(" <ID> ")" (* 空集检查 *)
```

## 示例代码1
```
{
    int val;
    set s;
    s={1,2,3};
    read val;
    if (val in s) {
        add s 10;
        write val;
    }
    while (val>0) {
        val = val -1;
        write val;
    }
    write 999;
}
```

## 示例代码2
```
{
    int x;
    set s;
    x=10;
    s={1,2};
    {
        set x; // 合法：遮蔽外层int x
        x={5,6}; // 此时x是集合
        add x 7;
        write x; // 输出{5,6,7}
        write s; // 合法：内层未声明s，向上查找外层s
    }
    write x; // 输出10：内层作用域销毁，x恢复为int
}
```

## 类P-code虚拟机指令系统

### 过程调用相关指令
| 指令格式 | 指令功能 |
| --- | --- |
| `INT 0 A` | 在栈顶开辟 A 个存储单元 |
| `OPR 0 0` | 结束被调用过程，返回调用点并退栈 |
| `CAL L A` | 调用地址为 A 的过程，调用过程与被调用过程的层差为 L |

### 存取指令
| 指令格式 | 指令功能 |
| --- | --- |
| `LIT 0 A` | 立即数 A 取到栈顶，t 加 1 |
| `LOD L A` | 将层差为 L、偏移量为 A 的存储单元的值取到栈顶，t 加 1 |
| `STO L A` | 将栈顶的值存入层差为 L、偏移量为 A 的单元，t 减 1 |

### 一元运算和比较指令
| 指令格式 | 指令功能 |
| --- | --- |
| `OPR 0 1` | 求栈顶元素的相反数，结果值留在栈顶 |
| `OPR 0 6` | 栈顶内容若为奇数则变为 1，若为偶数则变为 0 |

### 二元运算指令
| 指令格式 | 指令功能 |
| --- | --- |
| `OPR 0 2` | 次栈顶与栈顶的值相加，结果存入次栈顶，t 减 1 |
| `OPR 0 3` | 次栈顶的值减去栈顶的值，结果存放次栈顶，t 减 1 |
| `OPR 0 4` | 次栈顶的值乘以栈顶的值，结果存放次栈顶，t 减 1 |
| `OPR 0 5` | 次栈顶的值除以栈顶的值，结果存放次栈顶，t 减 1 |

### 二元比较指令
| 指令格式 | 指令功能 |
| --- | --- |
| `OPR 0 8` | 次栈顶与栈顶内容若相等，则将 1 存于次栈顶（否则存 0），t 减 1 |
| `OPR 0 9` | 次栈顶与栈顶内容若不相等，则将 1 存于次栈顶（否则存 0），t 减 1 |
| `OPR 0 10` | 次栈顶内容若小于栈顶，则将 1 存于次栈顶（否则存 0），t 减 1 |
| `OPR 0 11` | 次栈顶内容若大于等于栈顶，则将 1 存于次栈顶（否则存 0），t 减 1 |
| `OPR 0 12` | 次栈顶内容若大于栈顶，则将 1 存于次栈顶（否则存 0），t 减 1 |
| `OPR 0 13` | 次栈顶内容若小于等于栈顶，则将 1 存于次栈顶（否则存 0），t 减 1 |

### 转移指令
| 指令格式 | 指令功能 |
| --- | --- |
| `JMP 0 A` | 无条件转移至地址 A |
| `JPC 0 A` | 若栈顶为 0，则转移至地址 A，t 减 1 |

### 输入输出指令
| 指令格式 | 指令功能 |
| --- | --- |
| `OPR 0 14` | 栈顶的值输出至控制台屏幕，t 减 1 |
| `OPR 0 15` | 控制台屏幕输出一个换行 |
| `OPR 0 16` | 从控制台读入一行输入，置入栈顶，t 加 1 |


## 设计决策

| # | 决策项 | 结论 |
| --- | --- | --- |
| 1 | 编译器实现方式 | **纯 C 手工实现**（界面除外）。 |
| 2 | 语法分析算法 | **手写 LR / LALR(1) 表驱动分析器**。 |
| 3 | 交付形态 | **双产物，同一份 C 核心代码**：① 原生命令行版（主交付，保基本盘）；② Web 可视化演示版（Emscripten 编译为 wasm，加分项）。 |
| 4 | 编译目标 | **不可变**：L26 → 类 P-Code 指令 → 类 P-Code 虚拟机解释执行。wasm 只是"承载 C 虚拟机在浏览器运行"的打包方式，**L26 永不被编译成 wasm**，被编译成 wasm 的是工具链本身。 |
| 5 | LSP | **一核两壳**：前端分析做成干净、无 I/O 副作用、可被任意外壳调用的**诊断库**；原生 stdio JSON-RPC 壳与 wasm 浏览器内壳推迟到外壳阶段再选。 |
| 6 | union / inter | **仅两元**：一条集合表达式至多一个二元集合运算符，不支持链式 `s1 union s2 union s3`。 |
| 7 | read / write 类型 | **均不限类型**（int/bool/set 都可）。 |
| 8 | 承诺的附加功能 | **集合相等判定** + **集合推导式** + **Web 可视化演示**（显示 P-Code / 内存可视化 / 单步运行）。 |

## 架构

核心采用"一个分析核 + 多个外壳"的分层组织：

- **前端分析核**（纯 C，无 I/O 副作用，可被任意外壳调用）：词法扫描 → LR 语法分析 → 语义分析 / 符号表 → 生成 AST 与诊断信息。这是所有上层功能的共享地基。
- **代码生成器**：消费 AST，产出类 P-Code 指令序列。
- **诊断库**：同样消费前端分析核的诊断信息，供 LSP 复用——不重复实现解析。
- **P-Code 虚拟机**（纯 C）：解释执行 P-Code 指令，整数与集合统一在此运行。
- **外壳层**，全部调用上述纯 C 核心，互不重复逻辑：
  - *原生 CLI 运行壳*——命令行编译并运行（主交付）。
  - *wasm Web 页壳*——Emscripten 将核心编译为 wasm，页面负责 P-Code 展示、内存可视化、单步运行。
  - *原生 LSP 壳*——stdio JSON-RPC，供 VS Code/Neovim 等编辑器使用（推迟选型）。
  - *wasm LSP 壳*——浏览器内语言服务，与 Web 演示页融合（推迟选型）。

关键约束：前端、代码生成、虚拟机均为纯 C、无副作用，可同时被原生与 wasm 两类外壳链接，保证"核心逻辑只有一份"。

## 评分规则
### 基本功能
文档、程序、测试用例齐全；具备基本输入输出用户界面（界面部分可以使用其他语言编写）。
